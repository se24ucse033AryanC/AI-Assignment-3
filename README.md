# Part 1: Uninformed Search Techniques

Uninformed search algorithms explore the search space without using any heuristic information.

## Comparison of Uninformed Search Algorithms

| Algorithm | Complete | Optimal | Time Complexity | Space Complexity |
|------------|----------|----------|----------------|-----------------|
| Breadth First Search (BFS) | Yes | Yes (if equal cost) | O(b^d) | O(b^d) |
| Uniform Cost Search (UCS) | Yes | Yes | O(b^d) | O(b^d) |
| Depth First Search (DFS) | No | No | O(b^m) | O(bm) |
| Depth Limited Search (DLS) | No | No | O(b^l) | O(bl) |
| Iterative Deepening Search (IDS) | Yes | Yes | O(b^d) | O(bd) |
| Bidirectional Search | Yes | Yes | O(b^(d/2)) | O(b^(d/2)) |

Where:  
b = branching factor  
d = depth of optimal solution  
m = maximum depth  
l = depth limit

# Part 2: Informed Search Techniques

Informed search algorithms use heuristics to guide the search process.

## (i) Greedy Best First Search
- Uses heuristic function h(n)
- Expands node that appears closest to goal
- Not always optimal

## (ii) Best First Search
- Expands most promising node based on evaluation function
- Can combine cost and heuristic

## (iii) Dijkstra’s Algorithm
- Uses path cost g(n)
- No heuristic
- Always finds shortest path
- Used in road networks and routing systems

## (iv) A* Algorithm
- Uses f(n) = g(n) + h(n)
- Optimal if heuristic is admissible
- Widely used in robotics and games

## (v) Beam Search
- Limited memory version of BFS
- Expands only top k best nodes

## (vi) Recursive Best First Search (RBFS)
- Memory efficient version of A*
- Uses recursion and backtracking

## (vii) Iterative Deepening A* (IDA*)
- Combines A* and depth-first search
- Uses cost threshold

## (viii) Weighted A*
- Uses f(n) = g(n) + w*h(n)
- Faster but may not be optimal

# Heuristics

## Satisfying Search
A search that successfully reaches the goal state satisfying the problem constraints.

## Inadmissible Heuristic
A heuristic that overestimates the true cost to reach the goal. It may lead to non-optimal solutions.

## Designing a Heuristic
A heuristic can be designed by:
- Estimating distance to goal
- Using problem relaxation
- Using domain knowledge

Example: Manhattan distance for grid navigation.

## Generating Heuristics from Subproblems
Heuristics can be generated by:
- Solving a simplified version of the problem
- Ignoring constraints
- Combining multiple relaxed problem solutions

## 3. UGV Navigation – Dynamic Obstacles

In real battlefield environments, obstacles may appear dynamically and are not known beforehand.

To handle this, A* algorithm with real-time replanning is proposed.

### Approach:

1. Initialize grid with partially known environment.
2. Run A* to compute shortest path.
3. Move step-by-step toward goal.
4. Detect new obstacles using onboard sensors.
5. If path becomes blocked, re-run A* from current position.
6. Continue until goal is reached.

### Measures of Effectiveness:
- Path length
- Number of replans
- Computation time
- Memory usage
- Success rate

This approach ensures adaptive and optimal navigation in dynamic environments.
